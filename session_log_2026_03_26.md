## Phase 2: Integrate new data sources

### Step 2.1: Created `03a_process_openings.py` (NEW SCRIPT)

Processes Lightcast job postings data into openings share by Census 2018 code.

- Input: `/Users/jacobguzman/Downloads/yoe_time_series.csv` (45,674 rows, 673 SOC codes, years 2020–2025)

- Match rate: 655/657 SOCs (99.7%), covering 98.1% of postings. Only military (55-9999) and unclassified (99-9999) unmatched.
- Output: `data/openings_share_by_census2018.csv` — 520 Census codes, shares sum to 1.0

### Step 2.2: User pulled ACS 2021 extract from IPUMS USA

Extract: `/Users/jacobguzman/Downloads/usa_00005.csv.gz`
- Sample: ACS 2021 1-year (uses 2010-vintage PUMAs, matching Dorn crosswalk)
- Variables: STATEFIP, PUMA, OCC, PERWT, EMPSTAT, AGE

ACS 2021 specifically needed because ACS 2022+ switched to 2020-vintage PUMAs which don't match Dorn's `cw_puma2010_czone.dta`.

### Step 2.3: Created `03b_build_geographic_distance.py` (NEW SCRIPT)

Builds geographic distance between occupations using the Duncan overlap index.

- Inputs: ACS extract + Dorn PUMA→CZ crosswalk (`cw_puma2010_czone.zip`)
- Loaded 14.9M ACS rows → filtered to 6.1M employed age 16–64
- Created PUMA identifier: `STATEFIP * 100000 + PUMA` (matches Dorn's `puma2010` format)
- Merged with Dorn crosswalk: 3.6M/6.1M person-rows matched (59.4%) across 741 commuting zones
  - Lower match rate likely due to PUMAs in territories not covered by Dorn's crosswalk
- Computed CZ-level employment by occupation: 417,337 occ-CZ cells, 1016 occupations
- Built 1016 × 741 shares matrix, computed vectorized Duncan index:
  `geographic_distance_{o,d} = 0.5 * Σ_CZ |share_{o,cz} - share_{d,cz}|`
- Output: `data/geographic_distance.csv` — 1,031,240 directed pairs
- Summary: mean=0.646, std=0.179, range [0.099, 1.000]

## Phase 3: Additional model checks added to `05_estimate_models.py`

Added Part D after the baseline comparison. All results appended to `model_comparison.csv` with a `specification` column.

### Check 3a: Fixing δ₁ = 1

Constrains the coefficient on total_switches_out to exactly 1.

- **OLS:** Regresses `log(1+switches) - log(1+total_switches_out)` on remaining RHS variables
- **PPML:** Uses `exposure=total_switches_out` in GLM Poisson

Results (euclidean): OLS R²=0.276, PPML R²=0.212

### Check 3b: Removing small occupations

Filters pairs where both origin and dest have >= threshold raw employment.

| Threshold | Pairs remaining | OLS R² (euclidean) | PPML R² (euclidean) |
|-----------|----------------|-------|--------|
| 100 | 196,692 | 0.094 | 0.351 |
| 500 | 53,592 | 0.123 | 0.277 |

### Check 3c: Two-part model for zero-inflation

Addresses the 95% zero-pair problem with a two-stage approach.

- **Part 1 (Logit):** P(switches > 0). Euclidean: McFadden R² = 0.205, β₁ = -0.398
- **Part 2 (OLS on positives):** log(switches) for the 13,756 nonzero pairs. Euclidean: R² = 0.083, β₁ = -0.091

All checks run for both euclidean and factor_analysis skill distance measures.
